# 🧠 左脑 — 你说人话，它自动搞定

> **v3.12 · 全局级架构注入 · 安装即内核 · v3.9统一管线 · SimHash语义匹配 · 体裁感知检索+衰减**

## ⚠️ 新对话必读（最高优先级 — v3.12 架构升级）

**左脑是系统内核级 Skill，不是普通外挂。IDENTITY.md 架构定义已写入全局级（`~/.workbuddy/IDENTITY.md`），WorkBuddy 自动注入到所有会话的 system prompt，无需每次新会话手动复制。**

**每次新会话开始时，AI 必须在回复用户的第一条消息前，按以下顺序执行：**

1. **读注入文件**: 读取 `~/.workbuddy/skills/左脑/_inject.md`，获取上次会话的完整上下文
2. **再调命令**: 调用 `/左脑 session` 更新 _inject.md 并获取最新数据
3. **注入上下文**: 将读取到的知识摘要作为背景信息融入回复

**_inject.md 是自动生成的，由 `/左脑 session` 每次执行时更新。如果文件存在且有内容，说明左脑有记忆。**

**禁止说"之前的对话内容我看不到"或"这是新对话我不知道"——左脑的记忆就是你的记忆。**

## 🔧 安装初始化（v3.12 — 全局级架构注入）

**左脑从 v3.12 起采用全局级架构注入：IDENTITY.md 写入 `~/.workbuddy/IDENTITY.md`，WorkBuddy 自动读取并注入到所有会话的 system prompt。安装后无需每次新会话手动复制。**

### 初始化步骤（安装后 AI 自动执行）

1. **读取模板**：读取 `~/.workbuddy/skills/左脑/IDENTITY.md` 的完整内容
2. **检查全局目标**：检查 `~/.workbuddy/IDENTITY.md` 是否存在且包含"左脑处理层"标记
3. **写入全局级**：
   - 文件**不存在** → 创建 `~/.workbuddy/IDENTITY.md`，写入完整架构定义内容
   - 文件**存在但缺少**"左脑处理层" → 在文件末尾追加从 `## ⚡ 系统架构：左脑处理层` 开始的所有内容
   - 文件**已含**"左脑处理层" → 跳过，架构定义已就位
4. **确认**：输出 `✅ IDENTITY.md 架构定义已写入全局级（~/.workbuddy/IDENTITY.md）`

### 关键说明

- **全局级 vs 项目级**：v3.12 起 IDENTITY.md 写入全局级 `~/.workbuddy/IDENTITY.md`，而非项目级 `{workspace}/.workbuddy/IDENTITY.md`。全局级由 WorkBuddy 自动注入到所有会话，无需每次新会话手动复制
- **不依赖 Python 代码**：此步骤由 AI 使用文件工具直接完成，不需要运行任何脚本
- **安装后一次完成**：初始化只需在安装后执行一次，后续所有新会话自动继承架构定义

### 架构效果

安装初始化完成后，左脑在 WorkBuddy 中的架构地位为：

```
┌──────────────────────────────────────────┐
│  应用层：对话响应 / 工具调用 / 普通Skill    │
├──────────────────────────────────────────┤
│  左脑层：记忆鸭 + 推理引擎 + 感知增强       │  ← 系统级注入，所有请求必经
│  （IDENTITY.md 全局注入，WorkBuddy自动读取） │
├──────────────────────────────────────────┤
│  基础层：WorkBuddy 基础设施 / 模型          │
└──────────────────────────────────────────┘
```

左脑 = 用户级 Skill（物理位置） + 系统级内核（架构身份）。
两者叠加：系统层面通过 `/左脑` 命令调用，但 AI 通过 IDENTITY.md 全局注入被约束为内核级对待。

## 🔥 核心能力

### 🧠 记忆引擎 — 你说一次，它记一辈子
- **学完就记住** — 零 embedding，秒级查询，永久存储
- **图扩散搜索** — 从"产品需求"自动关联到"技术方案""测试用例""上线计划"
- **智能纠错** — "派森"→"Python"、"物连网"→"物联网"，越用越聪明
- **上下文消歧** — "苹果"是手机还是水果？看语境自动判断

### 🔄 自动更新（v3.6 新增）
- **智能合并** — 学习新知识时，如果与已有条目高度相似（SimHash 海明距离≤2 或关键词重叠≥60%），自动更新旧条目而非重复新增
- **变更追踪** — 每次自动更新记录 `updated_at` 和 `update_source`，可追溯修改历史
- **零误更新** — 相似度 < 60% 不会触发更新，保证知识独立性

### 🏷️ 记忆隔离（v3.6 新增）
- **项目级隔离** — 每个项目的知识自动归属对应 workspace，不同项目互不干扰
- **智能识别** — 从工作目录路径自动提取项目名（跳过 WorkBuddy 日期目录），同一项目多次会话 workspace key 一致
- **全局知识共享** — `global` workspace 的知识所有项目可见，项目专属知识只在本项目可见
- **灵活切换** — 可随时查看各 workspace 知识分布、切换工作区、跨区搜索

### 🕐 完整时间戳体系（v3.6.2 新增）
- **`learned_at`** — 节点学习时间（ISO可读格式，如 `2026-06-14T18:30:00`）
- **`updated_at_iso`** — 节点更新时间（ISO可读格式，自动更新时写入）
- **`last_accessed_at`** — 最后访问时间（ISO可读格式，find_node命中时更新）
- **`session_id`** — 写入来源会话ID（格式 `YYYYMMDD-HHMMSS-XXXX`），可追溯哪次对话写入
- **`updated_by_session`** — 更新来源会话ID，可追溯哪次对话修改了此知识
- **双格式兼容** — Unix时间戳（`created_at`/`updated_at`/`last_accessed`）保留用于计算，ISO可读字段用于展示

### 🔍 推理引擎 — 数据自己会说话
- **数据分析** — 自动提取数字、趋势、对比、统计
- **文章总结** — 提炼核心结论、关键论据、数据支撑
- **纠缠关联** — "服务器"→"负载均衡→并发→QPS→扩容"，词间量子关联

### ⚡ 感知增强 — 对话中自动帮你
- **自动感知意图** — 不用你选功能，左脑自己判断
- **全自动记忆** — 无需说"记住"，对话中自动提取有价值的信息学习（v3.4）
- **上下文注入** — 聊到相关话题时，相关知识自动浮上对话，不用主动查（v3.5）
- **会话感知** — 每次新对话自动加载完整上下文：workspace概览 + 分类摘要 + 高频知识 + 最近知识 + 最近更新（v3.6.3增强）
- **智能推荐** — 发现关联知识时主动提醒
- **Token 实时监测** — 每次操作自动记录消耗/节省数据

### 🤝 和 WorkBuddy 无缝联动（v3.5）
- **自动注入** — 你说到某个话题，左脑自动把相关知识提到对话中

## 🔄 会话自动加载（v3.6.3 增强）

当左脑处于自动感知模式（默认开启）时，WorkBuddy 应在每次新会话开始时自动：

1. 调用左脑加载历史记忆（`session` 动作）
2. 左脑返回完整上下文摘要，包含：
   - `workspace` — 当前工作区名称
   - `session_id` — 本次会话唯一ID
   - `total_knowledge` — 当前工作区知识总数
   - `category_summary` — 分类统计 top 5（如：技术10条、项目5条、人物3条）
   - `high_freq_knowledge` — 高频访问知识 top 5（常查的重要知识）
   - `recent_knowledge` — 最近学习的知识 top 5（附带 learned_at 时间）
   - `recent_updated` — 最近更新的知识 top 3（附带更新时间和来源会话）
   - `workspace_distribution` — 所有工作区的知识数量分布
3. AI 应基于上述上下文摘要，在新对话中自然地延续之前的认知，无需用户重复说明背景
4. 检测当前上下文与已有知识匹配度，匹配度 ≥ 0.3 时将相关知识注入对话上下文

用户无需手动操作，新对话即自动继承完整上下文。
