## 🧠 和左脑知识库的关系（重要）

| 维度 | 快速加载（本系统） | 左脑知识库 |
|:-----|:------------------|:-----------|
| **存什么** | 完整快照（对话 + KB + 文件状态） | 1-2 句话 KB 摘要 |
| **何时用** | 会话开头加载 | AI 推理时 recall |
| **数据量** | 大（每个快照 0.1-1 MB） | 小（每条 KB 几百字节） |
| **保存触发** | 你手动（一句话命令） | AI 自动（auto-perceive） |
| **保留时长** | 永久（除非手动删） | 永久 |
| **适合存** | 会话状态、文件状态、对话历史 | 事实、决策、偏好、知识关联 |

**协同工作流**：
1. AI 在对话中学到新事实 → **自动存左脑 KB**
2. 会话结束前 → **手动跑快照**（save.js + backup-history.js）
3. 下次会话开头 → **加载快照**（拿到完整上下文）
4. AI 工作中 → **recall 查左脑**（跨会话知识）

**两者互不替代**：
- 没快照：下次开会话 AI 不知道你做了啥
- 没左脑：AI 学到的东西容易忘

## ❓ 常见问题

> ⚠️ 这些是 FAQ，**不是启动指令**，不要复制。

**Q1: 多久备份一次？**
- **save.js**：每次会话结束前（重要）
- **backup-history.js**：里程碑 + 每周一次（可选）
- 左脑 KB：**AI 自动**，不用手动

**Q2: 快照文件太多怎么办？**
按标签删：
```bash
# 看所有快照
ls .claude/snapshots/

# 删某个标签
rm .claude/snapshots/*切换前*.md
```

**Q3: 加载快照后 AI 还是不理解？**
试试**双加载**：
```
1. 加载快照（拿到完整上下文）
2. 跑: bash .claude/skills/left-brain/scripts/session-summary.sh load
   （加载会话摘要到左脑）
3. 跑: bash .claude/skills/left-brain/scripts/left-brain.sh recall "智能调度"
   （查相关 KB）
```

**Q4: 快照系统能替代左脑吗？**
**不能**。两者职责不同：
- 快照 = 会话层（短期，详尽）
- 左脑 = 知识层（长期，精炼）

**Q5: history.jsonl 在哪？**
- Windows: `C:/Users/Administrator/.claude/history.jsonl`
- 备份脚本会自动找
