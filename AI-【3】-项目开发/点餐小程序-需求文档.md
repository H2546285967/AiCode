## 一、项目概述

### 1.1 产品定位
一个纯前端的微信点餐小程序，用户可以浏览菜单、选择菜品、下单、查看订单状态。
**不连接真实后端和数据库**，所有数据使用 Mock + 本地存储。

### 1.2 目标用户
- 餐厅顾客（通过微信扫码进入）
- 年龄：18-50 岁
- 设备：微信客户端（iOS + Android）

### 1.3 核心价值
- 扫码即用，无需注册登录
- 浏览菜单、加入购物车、一键下单
- 订单记录本地保存，可随时查看

## 二、技术约束（必须严格遵守）

### 2.1 技术栈
- **平台**: 微信小程序原生开发（不用 uni-app / Taro / mpvue）
- **语言**: JavaScript（不用 TypeScript）
- **UI**: 手写 WXSS，不用 UI 组件库（如 Vant Weapp）
- **后端**: 无后端，纯前端实现
- **数据存储**: wx.setStorage（本地存储）
- **数据来源**: Mock 数据（代码内硬编码菜单数据）

### 2.2 绝对不能做的事
- ❌ 不要用 TypeScript
- ❌ 不要引入 npm 依赖（不执行 npm install）
- ❌ 不要使用云开发（wx.cloud）
- ❌ 不要请求任何外部 API（全部 Mock）
- ❌ 不要使用任何第三方 UI 组件库
- ❌ 不要创建后端服务（无 Node.js / Python / Java 后端）

## 四、页面功能详细设计

### 4.1 菜单页（pages/menu/ — 首页，TabBar 第一项）

**布局：**
- 顶部：餐厅名称「美味小馆」+ 搜索框（纯展示，不需要真实搜索）
- 左侧：分类导航栏（竖排，点击切换分类）
- 右侧：菜品列表（当前分类下的菜品卡片）
- 底部：TabBar（菜单 / 购物车 / 订单）

**交互：**
- 点击左侧分类 → 右侧菜品列表滚动到对应分类
- 菜品卡片显示：emoji 图片 + 菜名 + 价格 + 描述 + 标签
- 每个菜品卡片右下角有「+」按钮，点击加入购物车
- 已加入购物车的菜品显示数量（- 1 + 按钮）

**TabBar 配置（app.json）：**
```json
{
  "tabBar": {
    "color": "#999999",
    "selectedColor": "#ff6b35",
    "list": [
      { "pagePath": "pages/menu/menu", "text": "菜单", "iconPath": "...", "selectedIconPath": "..." },
      { "pagePath": "pages/cart/cart", "text": "购物车", "iconPath": "...", "selectedIconPath": "..." },
      { "pagePath": "pages/orders/orders", "text": "订单", "iconPath": "...", "selectedIconPath": "..." }
    ]
  }
}
```
> TabBar 图标用纯色 PNG 占位图（1x1 像素），或用自定义 TabBar。

### 4.2 购物车页（pages/cart/）

**布局：**
- 顶部标题：「购物车」
- 购物车列表：每个条目显示 emoji + 菜名 + 单价 + 数量调整按钮（- 1 +）+ 小计
- 底部栏：合计金额 + 「去结算」按钮
- 空状态：居中显示「购物车空空如也，去菜单看看吧」

**交互：**
- 点击 +/- 调整数量，数量为 0 时自动移除
- 底部合计实时更新
- 点击「去结算」→ 跳转订单确认页
- 左滑条目可删除

### 4.3 订单确认页（pages/order/）

**布局：**
- 顶部标题：「确认订单」
- 订单详情：菜品列表 + 数量 + 单价
- 备注输入框（textarea）
- 合计金额
- 「提交订单」按钮

**交互：**
- 点击「提交订单」→ 生成订单对象 → 存入 wx.setStorage → 清空购物车 → 跳转到订单记录页
- 订单状态默认为「已提交」
- 生成订单号：`ORD` + 时间戳

### 4.4 订单记录页（pages/orders/）

**布局：**
- 顶部标题：「我的订单」
- 订单列表：每个订单显示订单号 + 时间 + 菜品摘要 + 合计金额 + 状态标签
- 空状态：「暂无订单，去点餐吧」

**交互：**
- 按时间倒序排列
- 状态标签颜色：已提交（蓝色）/ 制作中（橙色）/ 已完成（绿色）
- 点击订单可展开查看详情

**订单状态流转（自动模拟）：**
- 提交后 3 秒自动变为「制作中」
- 制作中 5 秒后自动变为「已完成」
- 使用 setTimeout 模拟（不需要真实定时器，进入页面时检查时间差即可）

## 六、组件设计

### 6.1 dish-card（菜品卡片）
- Props: `dish`（菜品对象）、`cartCount`（购物车中数量）
- Events: `add-to-cart`（加入购物车）
- 显示：emoji + 菜名 + 价格 + 描述 + 标签 + 辣度 + 加购按钮

### 6.2 cart-item（购物车条目）
- Props: `item`（含 dish 信息 + quantity）
- Events: `update-quantity`（±数量）、`remove`（删除）
- 显示：emoji + 菜名 + 单价 × 数量 = 小计

### 6.3 order-card（订单卡片）
- Props: `order`（订单对象）
- 显示：订单号 + 时间 + 菜品列表摘要 + 合计 + 状态标签
