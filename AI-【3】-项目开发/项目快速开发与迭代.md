## 一、新建项目

### 方式 A：Claude Code 内（推荐）
```
/new-project
```
按提示输入项目名、类型、需求文档路径。

### 方式 B：Bash 脚本
```bash
cd /path/to/workspace
# 仅脚手架（零 AI 成本）
bash .automation/new-project.sh <项目名> -t <类型>
# 脚手架 + 自动开发
bash .automation/new-project.sh <项目名> -t <类型> -r <需求文档> -d
```

**参数速查：**

| 参数 | 说明 | 示例 |
|:-----|:-----|:-----|
| `<项目名>` | 必填，英文+数字+连字符 | `my-api` |
| `-t` | 项目类型 | `springboot` |
| `-r` | 需求文档路径 | `requirements.md` |
| `-d` | 自动调用 Claude 开发 | - |
| `--skip-git` | 不初始化 git | - |

**支持的项目类型：**

| 类型 | 技术栈 |
|:-----|:-------|
| `springboot` | Java 21 + Spring Boot 3.x + Maven |
| `springboot-ai` | Java 21 + Spring Boot 3.x + Spring AI Alibaba |
| `springboot-langchain` | Java 21 + Spring Boot 3.x + LangChain4j |
| `python` | Python 3.11+ + FastAPI / Flask |
| `nodejs` | Node.js 20+ + Express / NestJS |
| `plain` | 纯项目（小程序 / 前端 / 工具等） |

### 方式 C：手动开发
```bash
cd "AI-【3】-项目开发/<项目名>"
claude
> 读取 REQUIREMENTS.md，按照其中的要求完成开发。
```

## 二、迭代开发（增加需求）

### 场景 1：加新功能
```bash
cd "AI-【3】-项目开发/<项目名>"
claude
> 在现有项目基础上，增加 XXX 功能。
> 具体要求：1. ... 2. ... 3. ...
> 不要修改现有的 XXX 模块。
```

### 场景 2：需求文档驱动
```bash
# 创建新需求文档 REQUIREMENTS_V2.md
claude
> 读取 REQUIREMENTS_V2.md，这是第二版需求。
> 在现有代码基础上迭代开发，不要重写已有功能。
```

### 场景 3：修 Bug
```bash
claude
> 运行 mvn test 发现 XXXTest 失败，错误信息如下：
> （粘贴错误日志）
> 请定位原因并修复。
```

### 场景 4：重构优化
```bash
claude
> 当前 Controller 层代码重复较多，请重构提取公共方法。
> 遵循 DRY 原则，保持 API 不变。
```

## 三、提示词构建指南

### 需求文档 5 层结构

写需求文档 = 给 AI 的"产品 PRD + 技术方案"合体。记住这 5 层：

```
1. 项目定义    → 这是什么、给谁用
2. 功能清单    → P0/P1/P2 分优先级，每个功能写清描述
3. 技术约束    → 用什么技术、不能用什么（最关键）
4. 交互细节    → 页面长什么样、数据怎么存
5. 验收标准    → 可检查的 checklist
```

## 四、提示词质量对照表

| 维度 | ❌ 模糊（AI 会乱猜） | ✅ 具体（AI 精确执行） |
|:-----|:----------------------|:----------------------|
| 功能 | "做个用户系统" | "手机号+验证码登录，60秒倒计时，JWT token 存 localStorage" |
| 技术 | "用 Spring Boot" | "Java 21 + Spring Boot 3.2 + Maven + JPA + MySQL 8.0，不用 MyBatis" |
| 交互 | "有个列表页" | "首页展示 10 条/页，下拉刷新，上拉加载，左滑删除" |
| 数据 | "存一下数据" | "MySQL 表 users，字段：id(BIGINT主键)、phone(VARCHAR11)、created_at(DATETIME)" |
| 验收 | "能跑就行" | "mvn test 全部通过、3 个接口返回正确 JSON、前端页面无白屏" |

**写需求的 5 个要点：**
1. **功能要具体** — "做个聊天功能" → "用户输入文字，调用 API，SSE 流式显示回复，保留 10 条上下文"
2. **技术要约束** — "随便用什么都行" → "原生小程序，不用 TypeScript，不用 npm，不用云开发"
3. **交互要明确** — "有个列表页" → "首页展示会话列表，按时间排序，左滑删除，右上角 + 新建"
4. **数据要定义** — "存一下数据" → "wx.setStorage 存储，key 为 conversations，结构如下..."
5. **验收要可测** — "能用就行" → "3 个页面可跳转、Mock API 正常、本地存储持久化"

## 五、多工具协作

一个项目可同时用多个 AI 工具开发，所有工具共享同一套约定：

```
my-api/
├── CLAUDE.md          ← Claude Code 自动读取
├── .qoderrules         ← Qoder 自动读取
├── .cursorrules       ← Cursor 自动读取
├── .lingma/           ← 通义灵码自动读取
│   └── instructions.md
├── .qoderrules        ← Qoder 自动读取
├── .minimaxrc         ← MiniMax Code 自动读取
└── src/               ← 实际代码
```

协作流程：
1. 用 **Claude Code** 搭建骨架 + 核心逻辑
2. 用 **Cursor** 调整前端
3. 用 **通义灵码** 在 IDEA 中调试
4. 所有工具遵守同一个 `AI-ClaudeCode-最佳实践精简.md`

## 七、注意事项

1. **需求文档越详细，自动开发效果越好** — 模糊需求 = AI 猜测 = 不可控产出
2. **自动开发消耗 Token** — 简单项目建议手动开发
3. **迭代时保留需求文档** — 每个版本单独文件（REQUIREMENTS.md、V2.md），便于追溯
4. **git 管理** — 每次重要迭代后提交，方便回滚
5. **AI-ClaudeCode-最佳实践精简.md 是约定源头** — 改一处，所有工具、所有项目生效
