**症状**：`/autonomous on` 报告 ON 但磁盘文件**没生成**，跨会话丢失状态。

**根因**：`scripts/orchestrator/autonomous.js` 第 35 行
```js
const WORKSPACE_ROOT = path.join(__dirname, '..', '..', '..');
```
`__dirname` = `H:\AI-han\AiCode\scripts\orchestrator`，**三级** `..` 走到了工程**外** `H:\AI-han\`，文件实际写到 `H:\AI-han\.claude/skills/left-brain/memory/autonomous-state.json`（用户主目录）。`status` 报告 ON 是因为脚本能**自己读回自己写错位置的文件**——但其他模块（state-snapshot.js 找的路径是工程内那个）就拿不到了。

**对照**：`state-snapshot.js` 的 `__dirname` 是 `.claude/skills/left-brain/scripts/`，要 4 级 `..` 才到工程根，那个才是对的。

**修法**：三级改两级。

**回归测试**：`test-autonomous.js` 第 7 段加了 4 个路径断言（路径必须在工程内、CLI on 后磁盘可见）。

**教训**：写涉及 `__dirname` 的路径时，必须先 `console.log` 实际跑一下确认；写完跑测试不止测功能，也测**文件落点**。
