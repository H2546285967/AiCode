### 项目概况

- **项目规模**: 2399个Java文件，企业级微服务架构
- **技术栈**: Java 8 + Spring Boot 2.6.13 + 自研框架(sheno) + Camunda BPMN
- **项目定位**: 企业级AI智能体平台（云部署+本地部署双模式）

### 你的真实角色

**不是架构师，不是从零搭建者，而是：在已有架构上做应用层开发维护的Java工程师。**

### 你的具体工作

| 工作内容 | 具体说明 |
|:---------|:---------|
| SSE流式输出对接 | 大模型API返回SSE流 → 逐chunk解析 → 推送给前端，处理流式和下游节点的冲突 |
| 新增工作流节点 | 在Camunda BPMN引擎上新增自定义节点（如文本提取节点），内部调用大模型 |
| 修改现有节点功能 | 根据需求修改已有节点逻辑 |
| 节点间变量赋值 | 通过Camunda流程变量做节点间数据传递（变量复制、变量提取） |
| 对接算法接口 | 按接口文档调用NL2SQL/RAG等算法接口，组装参数、处理返回 |
| 跨团队Debug | 出问题时：提取CURL报文 → Postman复现 → 定位是应用侧还是算法侧问题 |
| 对话流维护 | 维护会话管理、消息存储、多轮对话等模块 |

### 技术栈详情

| 技术 | 具体使用 |
|:-----|:---------|
| 流式输出 | SSE（Server-Sent Events），不是WebFlux响应式 |
| 工作流引擎 | Camunda BPMN，不是自研引擎 |
| 数据传递 | 流程变量赋值（key-value），不是JSON序列化 |
| 模型调用 | 按算法团队接口文档调用，不是自己设计策略模式 |
| 问题定位 | CURL + Postman，跨团队协作排查 |

### 你在这个项目中接触过的节点类型

```
变量复制节点 ← 把一个变量的值赋给另一个
变量提取节点 ← 从复杂结果中提取特定字段赋给新变量
文件解析节点 ← 解析上传的文件内容
文本提取节点 ← 调用大模型提取文本信息（新增）
LLM节点      ← 调用大模型生成回答
分类器节点   ← 意图分类/路由
HTTP请求节点 ← 调用外部API
代码沙箱节点 ← 执行用户代码（GraalVM）
回复/输出节点 ← 返回结果给用户
```
