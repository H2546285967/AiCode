## 🔍 各类型详细解析

### **1️⃣ 知识问答 (app_type = 0)**

**核心特征**：
- 基于 RAG 检索增强生成
- 依赖知识库文档片段
- 适合 FAQ、文档问答场景

**数据流转**：
```
用户提问 → RAG 检索 (向量 + 关键词) → Rerank 重排序 → TopK 文档片段 → LLM 生成答案
```


**关键配置**：
- `agent_app_config` 表存储：
  - `llm_config`: 大模型配置
  - `prompt`: 系统提示词
  - `dataset`: 知识库 ID 列表
  - `file_upload`: 文件上传配置

- [`RagProviderAdapter`](file:///D:/工作资料/code/仓颉智能体/nlp-agent/agent-common/agent-rag-adapter/src/main/java/com/yundingtech/agent/adapter/provider/RagProviderAdapter.java#L16-L212) - RAG 检索适配

### **2️⃣ 工作流 (app_type = 1)**

**核心特征**：
- BPMN 流程定义
- 多节点串行/并行执行
- 支持工具调用、条件分支、文件解析等节点

**节点类型**：
- `DirectReplyService` - 直接回复
- `AgentLLMService` - 大模型节点
- `ToolWorkflowService` - 工具/子流程调用
- `FileAnalysisService` - 文件解析
- `LogicProcessService` - 条件分支

**执行流程**：
```
开始节点 → 输入参数解析 → 节点执行 → 变量传递 → 输出结果
                ↓
        SSE 流式推送 (思考 + 回答)
```


**关键表**：
- `agent_workflow_config`: 存储 BPMN 流程定义和节点配置
- `agent_app_version`: 版本管理（工作流需部署到 Worker）

**代码位置**：
- [`AgentBaseAbstractDelegate`](file:///D:/工作资料/code/仓颉智能体/nlp-agent/agent-worker/src/main/java/com/yundingtech/agent/work/modules/workflow/service/delegate/AgentBaseAbstractDelegate.java#L52-L487) - 节点执行基类
- [`AgentAppConfigServiceImpl.savePublish()`](file:///D:/工作资料/code/仓颉智能体/nlp-agent/agent-builder/agent-build-core/src/main/java/com/yundingtech/agent/build/modules/appconfig/service/impl/AgentAppConfigServiceImpl.java#L345-L434) - 发布时部署到 Worker

### **3️⃣ 对话流 (app_type = 3)**

**核心特征**：
- 多轮对话引导
- 问题分类和意图识别
- 适合客服对话、问诊等场景

**关键能力**：
- [`QuestionClassifyService`](file:///D:/工作资料/code/仓颉智能体/nlp-agent/agent-worker/src/main/java/com/yundingtech/agent/work/modules/workflow/service/delegate/QuestionClassifyService.java#L177-L245) - 问题分类器
- 支持 LLM 分类和规则分类
- 对话上下文管理

**与知识问答的区别**：
- 知识问答：单次问答，基于文档检索
- 对话流：多轮引导，主动提问收集信息

---

### **4️⃣ 智能问数 (app_type = 4)**

**核心特征**：
- NL2SQL (Natural Language to SQL)
- 数据库查询
- 数据分析和报表生成

**关键表**：
- `nl2sql_db_app`: 存储数据库连接和表结构映射
