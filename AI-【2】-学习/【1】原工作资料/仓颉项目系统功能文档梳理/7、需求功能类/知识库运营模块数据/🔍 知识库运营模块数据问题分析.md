## 🔍 知识库运营模块数据问题分析

根据您提供的截图和代码分析，**知识库运营下的 3 个模块一直没数据**的问题是真实存在的。我已经找到了问题的根本原因:

### 📊 问题定位

从截图可以看到，以下 3 个模块没有数据:
1. **检索数量趋势** (空白图表)
2. **昨日活跃度** (显示 0 人次)
3. **活跃度趋势** (空白图表)

而其他模块如"被检索文档排行"、"关联应用使用情况"是有数据的。

### 🐛 根本原因

**核心问题：数据源查询逻辑错误**

查看代码发现，这 3 个模块的数据来源存在严重问题:

#### 1. **检索数量趋势** - SQL 查询错误
```xml
<!-- AgentOperationMapper.xml 第 527-545 行 -->
<select id="selectSearchNumTrendList">
    select
        TO_CHAR(create_time, 'MM-dd') as searchTime,
        count(1) as searchNum
    from agent_chat_message,
        jsonb_array_elements(config_references) as elems(value)
    <where>
        value ->> 'code' = #{code} and role = 'assistant'
```


**问题**: 这个 SQL 查询从 `agent_chat_message` 表的 `config_references` JSONB 字段中解析知识库检索记录，但查询条件 `value ->> 'code' = #{code}` 可能匹配不到数据，因为:
- `config_references` 字段可能为空或格式不匹配
- `code` 字段的匹配逻辑可能有问题

#### 2. **活跃度趋势** - 数据来源错误  

> 说明知识库操作就会如表base_Log

```java
// AgentOperationDashboardServiceImpl.java 第 1416-1428 行
QueryWrapper<AgentOperationKnowledgeEntity> queryWrapperActiveCountList = new QueryWrapper<>();
queryWrapperActiveCountList.select("stat_date", "active_count as count")
    .eq("knowledge_id", knowledgeId)
    .between("stat_date", startTime, endTime)
    .orderByAsc("stat_date");

List<Map<String, Object>> resultMaps = agentOperationKnowledgeMapper.selectMaps(queryWrapperActiveCountList);
```


**问题**: 活跃度数据从 `agent_operation_knowledge` 表查询，但这个表的数据依赖定时任务统计:
```java
// 定时任务 cron: 0 10 11 * * ? - 每天 11:10 执行
@Scheduled(cron = "0 10 11 * * ?")
public void statOperationKnowledegeData() {
    // 统计 T+1 数据
}
```


如果定时任务没有正确执行或统计逻辑有问题，就会导致没有数据。

#### 3. **昨日活跃度** - 同样依赖定时任务数据
```java
// AgentOperationDashboardServiceImpl.java 第 1383-1392 行

Integer activeNum = agentOperationKnowledgeEntity != null ? agentOperationKnowledgeEntity.getActiveCount() : 0;
```

### ✅ 解决方案

建议您按以下步骤排查和修复:

#### 方案 1: 检查定时任务是否正常执行
```sql
-- 查询 agent_operation_knowledge 表是否有数据
SELECT stat_date, knowledge_id, retrieval_count, hit_count, active_count 
FROM agent_operation_knowledge 
WHERE knowledge_id = '您的知识库 ID'
ORDER BY stat_date DESC 
LIMIT 10;
```


如果表为空，说明定时任务没有正确执行。

#### 方案 2: 手动触发数据统计
在 Controller 中已经有手动触发接口:
```
POST /api/operation/manual/statKnowledgeData
```


或者在代码中直接调用:
```java
agentOperationDashboardService.statOperationKnowledegeData();
```


#### 方案 3: 检查 SQL 查询逻辑
查看 `selectSearchNumTrendList` 的 SQL 是否能正确查询到数据:
```sql
-- 测试 SQL
SELECT 
    TO_CHAR(create_time, 'MM-dd') as searchTime,
    count(1) as searchNum
FROM agent_chat_message,
    jsonb_array_elements(config_references) as elems(value)
WHERE value ->> 'code' = '您的知识库 code' 
    AND role = 'assistant'
GROUP BY searchTime
ORDER BY searchTime;
```


#### 方案 4: 检查知识库 Code 是否正确获取
```java
// 第 1271 行
```
