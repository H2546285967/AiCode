# 🎯 向量库接口时间字段添加规范

## 结论先行

| 接口类型                   | 是否需要时间字段 | 说明                               |
| :------------------------- | :--------------- | :--------------------------------- |
| **新增/插入/上传数据接口** | ✅ **必须加**     | 需要 `create_time` / `update_time` |
| **删除/查询/集合操作接口** | ❌ **不需要加**   | 不写入新数据                       |

## 四、时间字段规范建议

### 统一使用以下两个字段：

| 字段名        | 类型               | 说明                     |
| :------------ | :----------------- | :----------------------- |
| `create_time` | 毫秒时间戳（Long） | 新增时写入，**永不修改** |
| `update_time` | 毫秒时间戳（Long） | 新增/修改时**都更新**    |

### 示例值：

```json
{
    "create_time": 1744198764000,
    "update_time": 1744198764000
}
```

### 代码示例：

```java
// 新增时
long now = System.currentTimeMillis();
entity.setCreateTime(now);
entity.setUpdateTime(now);

// 更新时
entity.setUpdateTime(System.currentTimeMillis());
```

## 五、为什么要加时间字段？

### 核心价值：**支持按时间清理历史数据**

```json
// 删除 30 天前的数据
{
    "filter": "create_time < 1712345678000"
}
```

### 典型应用场景：

| 场景             | 过滤条件                           |
| :--------------- | :--------------------------------- |
| 清理 7 天前数据  | `create_time < 当前时间 - 7天`     |
| 查询今日新增     | `create_time >= 今日0点`           |
| 查询最近更新     | `update_time >= 当前时间 - 24小时` |
| 数据生命周期管理 | 按 `create_time` 分批删除          |

## 六、接口入参/出参规范示例

### 示例1：新增接口（/repo/chunk_add）

**请求体**：
```json
{
    "data": [
        {

            "metadata": {
```

```json
    "message": "success",
```

## 八、注意事项

| 注意点       | 说明                                          |
| :----------- | :-------------------------------------------- |
| **时区**     | 统一使用 UTC+8（北京时间）                    |
| **精度**     | 毫秒级时间戳（13位数字）                      |
| **字段类型** | 数据库建议用 `bigint` 存储                    |
| **索引**     | 建议为 `create_time` 建立索引，便于按时间过滤 |
| **兼容性**   | 老数据没有时间字段，可默认填充 `0` 或 `null`  |
