public void handleVariables(String input, ...) {
    Pattern pattern = Pattern.compile("\\$\\{([^}]*)\\}");
    Matcher matcher = pattern.matcher(input);
    String[] parts = input.split("\\$\\{[^}]*}");
    
    int i = 0;
    while (matcher.find()) {
        // 1. 推送静态文本
        if (parts.length > i) {
            String real = parts[i];
            WorkerStreamResponseV1 streamResponseV1 = new WorkerStreamResponseV1(...);
            this.push(null, streamResponseV1, delegateExecution);  // 第 127 行
            contentBuilder.append(real);
        }
        
        String varName = matcher.group(1);
        
        // 2. 处理普通变量
        if (!varName.endsWith("llmText")) {
            String realValue = getVariableString(delegateExecution, varName);
            WorkerStreamResponseV1 streamResponseV1 = new WorkerStreamResponseV1(...);
            SseEmitter emitter = SseFactory.getSse(delegateExecution.getBusinessKey());
            emitter.send(SseEmitter.event().data(streamResponseV1));  // 第 146 行

        else {

private void handleLlmMessage(String businessKey, String sessionId, StringBuilder contentBuilder, ...) {
    // 1. 等待流开始
    long startTime = System.currentTimeMillis();
    long timeoutMillis = 50000;
    while (!streamMessageService.isStreaming(businessKey, varname)) {
        if (System.currentTimeMillis() - startTime > timeoutMillis) {
            log.warn("等待流开始超时");
            break;
        }
        Thread.sleep(50);

    // 2. 从 Redis 读取所有 chunks
    long count = 0L;
    while (streamMessageService.isStreaming(businessKey, varname)) {
        List<String> chunks = streamMessageService.getAllChunks(businessKey, varname, count);
        count = count + chunks.size();
        for (String chunk : chunks) {

        }

## 五、关键设计决策

### ✅ **为什么大模型节点要直接推送？**

**原因 1：流式体验要求**
- 打字机效果需要**实时推送**
- 不能等待后续节点处理完再推送

**原因 2：职责分离**
- 大模型节点：**生产内容 + 实时展示**
- 回复节点：**记录消息 + 最终归档**

**原因 3：避免重复**
- 如果回复节点也推送，前端会收到**两份相同内容**

---

### ✅ **为什么输出节点在对话流中不推送？**

```java
// OutputService 第 34-37 行
if (!"chatflow".equals(type)) {
    // 只在 workflow 场景推送
    WorkflowUtil.sendSseMessage(delegateExecution, outputList, ...);

```


**原因**：
- 大模型节点已经推送过了
- 输出节点只负责**变量替换和格式化**
- 推送职责已上移到大模型节点

---

### ✅ **为什么回复节点不推送？**

```java
// DirectReplyService 第 214-215 行

// this.push(null, streamResponseV1, delegateExecution); // 每条都推
```


**原因**：
- **避免重复推送**（大模型节点已推）
- 回复节点的职责是：
  - ✅ 从 Redis 读取数据
  - ✅ 拼装最终内容
  - ✅ 更新数据库记录
  - ❌ 不再推送到前端
