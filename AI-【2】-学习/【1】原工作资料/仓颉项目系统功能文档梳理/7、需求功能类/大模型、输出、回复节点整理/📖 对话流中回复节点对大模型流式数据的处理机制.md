# 📖 对话流中回复节点对大模型流式数据的处理机制

## 一、概述

在对话流（chatflow）场景中，当大模型节点（AgentLLmService）产生流式输出后，后续的回复节点（DirectReplyService）需要正确处理这些流式数据。本设计采用 **Redis 缓存 + 状态控制 + 防重复推送** 的机制，确保流式数据高效、准确地传输到前端。

**核心职责**：
- 调用大模型 API，获取流式响应（Flux<String>）
- 将流式数据块缓存到 Redis List
- 实时推送流式数据到前端（按 10 字符分块）

- 识别回复内容中的大模型变量（如 `${xxx_llmText}`）
- 从 Redis 读取流式数据并拼装完整内容
- **不重复推送**流式数据（因为 AgentLLmService 已推送）
- 更新数据库消息记录

**文件路径**：`D:\工作资料\code\仓颉智能体\nlp-agent\agent-worker\src\main\java\com\yundingtech\agent\work\modules\workflow\service\LlmRedisMessageService.java`

#### **步骤 2：缓存流式数据并实时推送**

```java
// 第 681-892 行：cacheStreamContent 方法
public void cacheStreamContent(Flux<String> flux, DelegateExecution execution, ...) {
    String businessKey = execution.getBusinessKey();
    
    // 1. 初始化 Redis 缓存
    streamMessageService.startStream(businessKey, getOutputContentKey(execution));
    
    StringBuilder contentBuilder = new StringBuilder();
    StringBuilder thinkBuilder = new StringBuilder();
    AtomicInteger lastPushedLength = new AtomicInteger(0); // 记录已推送长度

    // 2. 订阅流式数据
    flux.subscribe(
        chunk -> {
            // 2.1 添加到 Redis 缓存
            streamMessageService.addChunk(businessKey, getOutputContentKey(execution), chunk);
            
            // 2.2 解析 chunk，分离思考和回答内容
            if (!chunk.equals("[DONE]")) {
                handleMessage(chunk, new AtomicBoolean(false), contentBuilder, thinkBuilder, execution);
            }

            // 2.3 实时推送到前端（按 10 字符分块）
            int chunkSize = 10;
            String currentContent = contentBuilder.toString();
            int currentLength = currentContent.length();
            int start = lastPushedLength.get();

            if (currentLength >= start + chunkSize) {
                String newContent = currentContent.substring(start);
                while (newContent.length() >= chunkSize) {
                    String subContent = newContent.substring(0, chunkSize);
                    
                    WorkerStreamResponseV1 streamResponseV1 = new WorkerStreamResponseV1(
                        execution.getBusinessKey(), workflowId, sessionId,
                        "assistant", "", subContent, // think 为空，只推送回答内容分块
                        0, 0, 0, 0.0, null, getCurrentTimeStr());
                    
                    push(null, streamResponseV1, execution); // 推送到前端
                    lastPushedLength.addAndGet(chunkSize);
                    
                    newContent = newContent.substring(chunkSize);
                }
            }
        },
        throwable -> {
            // 异常处理
            streamMessageService.endStream(businessKey, getOutputContentKey(execution));
            isStop.put(businessKey, true);
            // ... 推送错误信息
        },
        () -> {
            // 完成回调
            streamMessageService.endStream(businessKey, getOutputContentKey(execution));
            
            // 发送剩余内容（不足 10 字符的部分）
            String remainingContent = currentContent.substring(lastPushedLengthValue);
            if (!remainingContent.isEmpty()) {
                WorkerStreamResponseV1 streamResponseV1 = new WorkerStreamResponseV1(
                    ..., remainingContent, ..., configReferences, ...);
                push(null, streamResponseV1, execution);
            }
            
            // 保存流程变量（llmText、llmThinkText）
            bpmProcVariableService.saveNewTx(bpmProcVariableEntities);
        }
    );
}
```

#### **步骤 2：从 Redis 读取流式数据**

```java
// 第 179-233 行：handleLlmMessage 方法
private void handleLlmMessage(String businessKey, String sessionId, ...) {
    // 1. 等待流开始（最多等 5 秒）
    long startTime = System.currentTimeMillis();
    long timeoutMillis = 50000;
    
    while (!streamMessageService.isStreaming(businessKey, varname)) {
        if (System.currentTimeMillis() - startTime > timeoutMillis) {
            log.warn("等待流开始超时");
            break;
        }
        Thread.sleep(50);
    }
    
    // 2. 循环读取直到流结束
    long count = 0L;
    long thinkStartTime = System.currentTimeMillis();
    long thinkEndTime = System.currentTimeMillis();
    
    while (streamMessageService.isStreaming(businessKey, varname)) {
        try {
            // 2.1 从 Redis 获取所有数据块
            List<String> chunks = streamMessageService.getAllChunks(businessKey, varname, count);
            count = count + chunks.size();
            
            // 2.2 处理每个 chunk
            for (String chunk : chunks) {
                WorkerStreamResponseV1 streamResponseV1 = handleMessage(chunk, businessKey, 
                    new AtomicBoolean(false), contentBuilder, thinkBuilder, 
                    sessionId, agentChatMessageEntity.getId(), delegateExecution, workflowId);
                
                if (streamResponseV1 != null) {
                    if (StringUtils.isNotBlank(streamResponseV1.getThink())) {
                        thinkEndTime = System.currentTimeMillis();
                    }
                    streamResponseV1.setTimes(Math.round((((double) thinkEndTime - thinkStartTime) / 1000) * 100.0) / 100.0);
                    
                    // ⚠️ 关键：不在此处推送！因为 AgentLLmService 已经推送了流式数据
                    // this.push(null, streamResponseV1, delegateExecution); // 注释掉，避免重复
                }
            }
            Thread.sleep(10); // 控制轮询频率
        } catch (InterruptedException e) {
            run = false;
            Thread.currentThread().interrupt();
            break;
        }
    }
    
    // 3. 设置流程变量
    delegateExecution.setVariable(AgentLLmService.OUTPUT_PARAM, contentBuilder.toString());
    WorkflowServiceImpl.setThinkTime(businessKey, (double) (thinkEndTime - thinkStartTime));
}
```

#### **步骤 3：解析 chunk 数据**

```java
// 第 423-473 行：handleMessage 方法
private WorkerStreamResponseV1 handleMessage(String chunk, String businessKey, 
        AtomicBoolean think, StringBuilder contentBuilder, StringBuilder thinkBuilder, ...) {
    
    StreamResponse streamResponse = JSONObject.parseObject(chunk, StreamResponse.class);
    List<StreamChoice> choices = streamResponse.getChoices();
    
    String thinkContent = "";
    String content = "";
    
    // 判断思考开始/结束
    if (chunk.contains("<think>")) {
        think.set(true);
        log.info("【思考过程】");
    } else if (chunk.contains("</think>")) {
        think.set(false);
        log.info("【思考过程结束】");
    } else {
        if (choices != null && !choices.isEmpty()) {
            for (StreamChoice streamChoice : choices) {
                String reasoningContent = streamChoice.getDelta().getString("reasoning_content");
                
                if (think.get() || !StringUtil.isNullOrEmpty(reasoningContent)) {
                    // 思考内容
                    thinkContent = reasoningContent;
                    thinkBuilder.append(thinkContent);
                } else {
                    // 回答内容
                    Object contentObj = streamChoice.getDelta().get("content");
                    if (contentObj == null) continue;
                    content = contentObj.toString();
                    contentBuilder.append(content);
                }
            }
        }
    }
    
    return new WorkerStreamResponseV1(..., thinkContent, content.replace("\n\n","\n").replace("\n","\n\n"), ...);
}
```

### 🔹 3.3 Redis 缓存管理（LlmRedisMessageService）

```java
@Component("streamMessageService")
public class LlmRedisMessageService {
    
    // 1. 核心数据结构：控制流是否还在进行中
    private static final Map<String, AtomicBoolean> isStreaming = new ConcurrentHashMap<>();
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    private String getRedisKey(String businessKey, String variable) {
        return "stream:" + businessKey + ":" + variable;
    }
    
    // 2. 添加流式数据块（List 右推）
    public void addChunk(String businessKey, String variable, String chunk) {
        redisTemplate.opsForList().rightPush(getRedisKey(businessKey, variable), chunk);
    }
    
    // 3. 获取所有数据块（支持分页读取）
    public List<String> getAllChunks(String businessKey, String variable, Long startIndex) {
        String key = getRedisKey(businessKey, variable);
        Long size = redisTemplate.opsForList().size(key);
        if (size == null || size == 0) {
            return Collections.emptyList();
        }
        return redisTemplate.opsForList().range(key, startIndex, -1);
    }
    
    // 4. 流状态控制
    public void startStream(String businessKey, String variable) {
        String key = getRedisKey(businessKey, variable);
        log.info("开始流处理：{}", key);
        isStreaming.put(key, new AtomicBoolean(true));
    }
    
    public void endStream(String businessKey, String variable) {
        String key = getRedisKey(businessKey, variable);
        log.info("结束流处理：{}", key);
        isStreaming.getOrDefault(key, new AtomicBoolean(false)).set(false);
    }
    
    public boolean isStreaming(String businessKey, String variable) {
        String key = getRedisKey(businessKey, variable);
        return isStreaming.getOrDefault(key, new AtomicBoolean(false)).get();
    }
    
    // 5. 清理资源
    public void cleanup(String businessKey, String variable) {
        String key = getRedisKey(businessKey, variable);
        log.info("清空流：{}", key);
        redisTemplate.delete(getRedisKey(businessKey, variable));
        isStreaming.remove(businessKey);
    }
}
```

### ✅ 2. 流式配置智能判断

**代码位置**：AgentLLmService 第 924-1021 行

```java
private Boolean checkNextNodeStreamConfig(DelegateExecution execution) {
    // 1. 获取当前节点和 BPMN 模型
    String currentActivityId = execution.getCurrentActivityId();
    FlowNode currentNode = (FlowNode) modelInstance.getModelElementById(currentActivityId);
    Collection<SequenceFlow> outgoingFlows = currentNode.getOutgoing();
    
    // 2. 获取下一节点
    SequenceFlow outgoingFlow = outgoingFlows.iterator().next();
    FlowElement targetElement = outgoingFlow.getTarget();
    
    // 3. 判断是否为 ServiceTask（输出节点）
    if (!(targetElement instanceof ServiceTask)) {
        return null;
    }
    
    // 4. 读取 camunda：inputParameter 中的 isStream 配置
    for (CamundaInputParameter inputParam : inputParameters) {
        if ("isStream".equals(inputParam.getCamundaName())) {
            String isStreamValue = inputParam.getTextContent();
            
            // 5. 支持变量引用 ${variableName}
            if (isStreamValue.startsWith("${") && isStreamValue.endsWith("}")) {
                String variableName = isStreamValue.substring(2， isStreamValue.length() - 1);
                Object variableValue = execution.getVariable(variableName);
                if (variableValue != null) {
                    return Boolean.parseBoolean(variableValue.toString());
                }
            }
            return Boolean.parseBoolean(isStreamValue);
        }
    }
    return null;
}
```

**决策逻辑**：
- 下一节点是输出节点且 `isStream = false` → 大模型采用 **非流式模式**
- 下一节点是输出节点且 `isStream = true` → 大模型采用 **流式模式**
- 下一节点不是输出节点 → 默认采用 **流式模式**

## 七、常见问题与注意事项

### ⚠️ 1. 避免 isStreaming 判断导致死循环
**经验记忆**：`避免因 isStreaming 状态判断导致死循环`

**正确做法**：

```java
// DirectReplyService 第 185-196 行

while (!streamMessageService.isStreaming(businessKey， varname)) {

        log.warn("等待流开始超时");
        break; // 超时退出，避免死循环
    }
    Thread.sleep(50);
}
```

### ⚠️ 2. 避免 fastjson 类型转换异常
**经验记忆**：`避免 fastjson 与 fastjson2 混用导致的类型转换异常`

**注意事项**：
- 使用 `fastjson2` 解析 `StreamResponse`
- 确保依赖版本一致
- 捕获 `ClassCastException` 异常

### ⚠️ 3. 大模型变量检查需在替换前进行
**经验记忆**：`OutputService 中大模型变量检查需在替换前进行`

**正确顺序**：
1. 先检查变量是否包含 `llmText`
2. 如果是大模型变量，走流式处理逻辑
3. 否则直接获取变量值并推送

### ⚠️ 4. 禁止大模型节点直接推送 nodeState
**经验记忆**：`对话流中禁止大模型节点直接推送 nodeState`

**原因**：
- nodeState 应由流程引擎统一管理
- 提前推送会导致状态混乱

## 十、总结

这套流式数据处理机制的核心设计理念：

1. **职责分离**：AgentLLmService 负责生产和推送，DirectReplyService 负责消费和记录
2. **Redis 缓存**：使用 Redis List 作为临时存储，解耦生产者和消费者
3. **状态控制**：通过 `isStreaming` 精确控制流的生命周期
4. **防重复推送**：明确分工，避免同一数据被多次推送到前端
5. **智能适配**：根据下一节点配置自动选择流式/非流式模式
6. **健壮性**：超时保护、异常处理、异步执行等多重保障

通过这些设计，确保了对话流中流式数据的高效、稳定传输！🎉
