## ⏰ 定时任务概览

### 三个定时任务

| 定时任务 | Cron 表达式 | 执行时间 | 统计方法 | 数据表 |
|---------|-----------|---------|---------|--------|
| **App 运营数据统计** | `0 5 0 * * ?` | 每天 0:05 | `statOperationAppData()` | `agent_operation_app` |
| **知识库运营数据统计** | `0 10 0 * * ?` | 每天 0:10 | `statOperationKnowledegeData()` | `agent_operation_knowledge` |
| **文件运营数据统计** | `0 15 0 * * ?` | 每天 0:15 | `statOperationFile()` | `agent_operation_file` |

### 统计模式：T+1

所有定时任务都采用 **T+1 统计模式**:
- **今天 (3 月 10 日)** 凌晨执行 → 统计 **昨天 (3 月 9 日)** 的数据
- 统计时间范围：`昨天 00:00:00` 到 `今天 00:00:00`

## 📊 统计逻辑详解

### 1️⃣ App 运营数据统计 (`statOperationAppData`)

#### 执行流程

```java
// 文件位置：AgentOperationDashboardServiceImpl.java:140-217

LocalDate today = LocalDate.now();           // 2026-03-10
LocalDate targetDate = today.minusDays(1);   // 2026-03-09
LocalDateTime startTime = targetDate.atStartOfDay();      // 2026-03-09 00:00:00
LocalDateTime endTime = targetDate.plusDays(1).atStartOfDay(); // 2026-03-10 00:00:00
```

#### 数据来源

| 数据字段 | 来源 SQL | 说明 |
|---------|---------|------|
| `app_session_count` | `getAppGroupedDailyStats()` | 从 `agent_chat_session` 表统计会话数 |
| `app_token_usage` | `getAppGroupedDailyStats()` | 从 `agent_chat_message` 表统计 tokens |
| `app_user_count_daily` | `getDailyUserCountByApp()` | 从 `agent_chat_message` 表统计去重用户数 |
| `app_like_count` | `getAppGroupedDailyStats()` | 从 `agent_chat_message_feedback` 表统计点赞 |
| `app_dislike_count` | `getAppGroupedDailyStats()` | 从 `agent_chat_message_feedback` 表点踩 |
| `hit_count` | `getAppGroupedDailyStats()` | 知识库命中数 |
| `repo_count` | `getAppGroupedDailyStats()` | 知识库检索数 |

#### 核心 SQL

**对话用户数统计 SQL:**
```sql
-- getDailyUserCountByApp
SELECT 
    app_id AS key1,
    DATE(create_time) AS "date",
    COUNT(DISTINCT create_user_id) AS "value"
FROM agent_chat_message
WHERE role = 'assistant'
  AND create_time BETWEEN #{startTime} AND #{endTime}
GROUP BY app_id, DATE(create_time)
```

**关键点:**
- 只统计 `role = 'assistant'` 的消息
- 按 `create_user_id` 去重
- 如果 `create_user_id` 为 NULL，则 `COUNT(DISTINCT NULL) = 0`

## 🔍 常见问题排查

### 问题 1: 某天对话用户数为 0

**可能原因:**
1. ✅ **当天确实没有用户对话** (最常见)
2. ⚠️ `create_user_id` 字段为 NULL
3. ⚠️ 时区问题导致统计日期偏移

**排查步骤:**

```sql
-- 1. 检查当天是否有对话记录
SELECT 
    COUNT(*) as total_messages,
    COUNT(DISTINCT create_user_id) as unique_users
FROM agent_chat_message
WHERE role = 'assistant'
  AND create_time >= '2026-03-09 00:00:00'
  AND create_time < '2026-03-10 00:00:00';

-- 2. 检查是否有 NULL 的 create_user_id
SELECT 
    COUNT(*) as null_user_messages
FROM agent_chat_message
WHERE role = 'assistant'
  AND (create_user_id IS NULL OR create_user_id = '')
  AND create_time >= '2026-03-09 00:00:00'
  AND create_time < '2026-03-10 00:00:00';

-- 3. 查看最新的对话时间
SELECT 
    MAX(create_time) as last_message_time,
    role
FROM agent_chat_message;
```

## 📝 手动统计接口使用说明

### 接口列表

| 接口 | 功能 | 请求方式 |

### 注意事项

1. ⚠️ **T+1 统计**: 所有接口都统计**昨天的数据**

3. ⚠️ **临时接口**: 验证完成后建议删除这些接口
4. ⚠️ **日志记录**: 所有操作都会记录日志

## 🎯 总结

### 核心要点

1. **定时任务执行时间:**
   - App 数据：每天 0:05
   - 知识库数据：每天 0:10
   - 文件数据：每天 0:15

2. **统计模式:**
   - T+1 模式 (统计前一天的数据)
   - 统计范围：`昨天 00:00:00` 到 `今天 00:00:00`

3. **数据来源:**
   - `agent_chat_message` - 对话消息表
   - `agent_chat_session` - 会话表
   - `agent_chat_message_feedback` - 反馈表
   - `base_log` - 日志表

4. **关键 SQL:**
   ```sql
   -- 对话用户数统计
   SELECT COUNT(DISTINCT create_user_id)
   FROM agent_chat_message
   WHERE role = 'assistant'
     AND create_time BETWEEN #{startTime} AND #{endTime}
   ```
