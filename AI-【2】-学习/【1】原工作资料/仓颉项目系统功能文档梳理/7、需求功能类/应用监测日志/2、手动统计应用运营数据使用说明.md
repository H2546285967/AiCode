# 手动统计应用运营数据 - 调用说明

## 📋 功能说明

提供了手动触发应用运营数据统计的功能，支持自定义日期范围，用于重新统计或补充历史数据。

## 🔧 实现内容

### 1. Service 接口方法

**文件路径**: `agent-builder/agent-build-core/src/main/java/com/yundingtech/agent/build/modules/operation/service/IAgentOperationDashboardService.java`

```java
/**
 * 手动统计应用运营数据（支持自定义日期范围）
 * @param startDate 开始日期
 * @param endDate 结束日期
 */
void statOperationAppDataManual(LocalDateTime startDate, LocalDateTime endTime);
```

### 2. Service 实现方法

**文件路径**: `agent-builder/agent-build-core/src/main/java/com/yundingtech/agent/build/modules/operation/service/impl/AgentOperationDashboardServiceImpl.java`

主要功能：
- 统计指定日期范围内的应用运营数据
- 包括：会话数、Token 消耗、用户数、点赞/点踩、知识库命中、运行成功率等
- 支持幂等操作（先删除再插入）
- 添加完整的日志记录

### 3. Controller HTTP 接口

**文件路径**: `agent-builder/agent-build-core/src/main/java/com/yundingtech/agent/build/modules/operation/controller/AgentOperationController.java`

```java
/**
 * 手动统计应用运营数据（支持自定义日期范围）
 *
 * @param startDate 开始日期（格式：yyyy-MM-dd）
 * @param endDate 结束日期（格式：yyyy-MM-dd）
 * @return CommonResult<Void>
 */
@GetMapping("/manual/statAppData")
@Operation(summary = "手动统计应用运营数据")
public CommonResult<Void> manualStatAppData(
        @RequestParam @DateTimeFormat(pattern = "yyyy-MM-dd") LocalDate startDate,
        @RequestParam @DateTimeFormat(pattern = "yyyy-MM-dd") LocalDate endDate)
```

### 方式二：直接调用 Service

在其它 Service 或 Controller 中注入后直接调用：

```java
@Autowired
private IAgentOperationDashboardService agentOperationDashboardService;

// 示例：统计 2026 年 3 月 1 日 到 3 月 7 日的数据
LocalDateTime startTime = LocalDateTime.of(2026, 3, 1, 0, 0, 0);
LocalDateTime endTime = LocalDateTime.of(2026, 3, 8, 0, 0, 0); // 注意：结束时间要加 1 天

agentOperationDashboardService.statOperationAppDataManual(startTime, endTime);
```

### 方式三：定时任务中调用

如果需要创建自定义的定时任务：

```java
@Component
public class CustomStatTask {
    
    @Autowired
    private IAgentOperationDashboardService agentOperationDashboardService;
    
    @Scheduled(cron = "0 0 2 * * ?")  // 每天凌晨 2 点执行
    public void executeStat() {
        // 统计最近 7 天的数据
        LocalDateTime endTime = LocalDateTime.now();
        LocalDateTime startTime = endTime.minusDays(7);
        
        agentOperationDashboardService.statOperationAppDataManual(startTime, endTime);
    }
}
```

## 📊 统计的数据内容

该方法会统计以下数据并写入 `agent_operation_app` 表：

1. **基础数据**
   - 会话数 (appSessionCount)
   - Token 消耗 (appTokenUsage)
   - 点赞数 (appLikeCount)
   - 点踩数 (appDislikeCount)

2. **用户数据**
   - 当日活跃用户数 (appUserCountDaily)
   - 累计用户数 (appUserCountAll)
   - 工作空间用户数 (workspaceUserCountDaily)

3. **知识库数据**
   - 检索次数 (repoCount)
   - 命中次数 (hitCount)

4. **运行数据**
   - 运行次数 (runCount)
   - 成功次数 (runSuccessCount)

## ⚠️ 注意事项

1. **时间范围**: 
   - 结束时间会自动加 1 天（包含结束日期当天）
   - 例如：startDate=2026-03-01, endDate=2026-03-07，实际统计的是 3 月 1 日到 3 月 7 日（共 7 天）

2. **幂等性**: 
   - 方法会先删除指定日期范围的旧数据，再插入新数据
   - 可以重复调用，不会产生重复数据

3. **事务**: 
   - 方法带有事务注解，失败会自动回滚
   - 统计大量数据时注意事务超时问题

4. **性能**: 
   - 统计大量历史数据时可能耗时较长
   - 建议在业务低峰期执行
   - 可以分批多次调用（每次统计几天的数据）

5. **日志**: 
   - 方法会记录详细的执行日志
   - 成功日志：`手动统计应用运营数据完成，统计天数：{count}`
   - 失败日志：`手动统计应用运营数据失败`

## 🔍 验证统计结果

统计完成后，可以通过以下 SQL 验证数据：

```sql
-- 查看指定日期的统计数据
SELECT 
    stat_date,
    app_id,
    app_session_count,
    app_token_usage,
    app_user_count_daily,
    repo_count,
    hit_count
FROM agent_operation_app
WHERE stat_date BETWEEN '2026-03-01' AND '2026-03-07'
ORDER BY stat_date, app_id;

-- 查看某应用的日活趋势
SELECT 
    stat_date,
    app_user_count_daily
FROM agent_operation_app
WHERE app_id = '799314226807067205'
ORDER BY stat_date;
```

## 💡 常见使用场景

1. **补充历史数据**: 系统上线后需要补充之前的运营数据
2. **数据修复**: 发现统计数据有误，需要重新统计
3. **测试验证**: 测试环境验证统计逻辑是否正确
4. **临时统计**: 需要统计特定时间段的数据进行分析
