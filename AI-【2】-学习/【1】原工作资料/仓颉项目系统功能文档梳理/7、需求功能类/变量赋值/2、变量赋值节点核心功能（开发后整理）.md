## 变量赋值节点核心功能

**将全局变量设置为新的值，供所有后续节点使用**

在对话流中，后续所有对话将使用新的值。

---

## 🔧 关键修复内容

我们解决了一个**节点变量无法传递给后续节点**的问题：

### 问题根源
- Camunda 在节点结束时会自动执行 `outputParameter` 映射
- 由于变量赋值节点没有配置 `outputParameter`，节点变量会被**自动置空**
- 后续节点从内存中读取节点变量时，得到的是空值

### 修复方案（2 个关键步骤）

**步骤 9.5：保存节点变量到数据库**
```java
saveVar(delegateExecution, nodeVarName, realValue, BpmConstant.NODE_END_TAG);
```


**步骤 9.6：禁用 outputParameter 自动映射** ⭐
```java
ExecutionEntity executionEntity = (ExecutionEntity) delegateExecution;
executionEntity.setSkipIoMappings(true);
```

## 📋 完整执行流程

```
步骤 1: 获取节点配置（从 BPMN 扩展属性）
步骤 2: 解析赋值配置列表（JSON 格式）
步骤 3: 遍历赋值列表
  ├─ 步骤 4: 解析变量值（处理 ${varName} 引用）
  ├─ 步骤 5: 类型转换（string/number/boolean 等）
  ├─ 步骤 6: 设置全局变量和节点变量（内存）
  ├─ 步骤 7: 保存全局变量到数据库
  ├─ 步骤 8: 不保存节点变量到数据库（向文本提取节点学习）
  ├─ 步骤 9: 显式设置 Camunda 节点变量
  ├─ 步骤 9.5: 保存节点变量到数据库（新增）
  ├─ 步骤 9.6: 禁用 outputParameter 自动映射（新增）⭐
  └─ 步骤 10: 同步更新开始节点的全局变量
```

## 💡 技术要点

1. **向文本提取节点学习**：只设置必要的变量
2. **使用 `setSkipIoMappings(true)`**：跳过 Camunda 的 inputOutput 自动映射
3. **三类变量同步更新**：
   - 全局变量：`varName`
   - 节点变量：`id_xxxGROUPvarName`
   - 开始节点全局变量：`globalGROUPvarName`

需求改动：

```

需求要求在同一个对话内这个全局变量被修改后获取的都是新的
```
