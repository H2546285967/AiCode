## 🧠 二、核心能力详解

### 2.1 智能调度（v1.2.0）

**核心问题**：复杂任务让 AI 一个个查很慢，能不能**自动派 2-3 个 Agent 并行**？

**解决方案**：抖音视频同款"主从并行加速"

```
你输入"排查 BUG"
       ↓
dispatcher.js 分析（规则引擎 12 项匹配）
       ↓
判断：派 2 个 Agent（后端 + 前端）
       ↓
并行执行（不互相等）
       ↓
主会话等所有完工 → 汇总 → 输出
```

**实测提速**：
- 串行：3-4 分钟
- 并行：1-2 分钟
- **提速 2-3 倍**

**11 项 v1.2 改进**：
- 灰区 `suggested_action` 字段
- 累加权重（多关键词叠加）
- 7 个中文口语词
- 钩子异常加 `errorType: 'crash'`
- `learn-rules` 简化为 `bad/good` 命令
- `RULES.version = v1.2.0`
- 英文 deploy/rollback/migrate detect
- `confidence` 改 0-1 数字
- 4 工具统一入口（index.js）
- `learn-rules` prompt hash 去重
- 钩子写日志

**使用方式**：
- **自动**：每次提问 PreToolUse 钩子自动跑
- **手动**：`/dispatch 排查 BUG`
- **强制**：`/parallel 3 排查 X`

### 2.2 快照系统（v1.1）

**核心问题**：会话太长 context 爆炸，下次开会话忘记上次的进度。

**解决方案**：一键备份 + 一秒恢复

```
# 结束会话前（一句话）
node scripts/会话快照/save.js "任务标题" "中文标签" -m "下一步"

# 下次开会话（一键）
node scripts/会话快照/load.js v1.1    # 按版本匹配
cat ROOT_快速加载会话.md                # 看根目录索引
```

**自动化**：
- save.js 自动追加 ROOT_快速加载会话.md 的"快照列表"表格
- 自动加 ⭐ 最新 标记
- 自动加"快速启动命令"代码块
- 检测"完成/里程碑/交付"标签 → 加三级检查点提示

**完整对话备份**：
```bash
node scripts/会话快照/backup-history.js "标签"
# 把 CC 的 history.jsonl（0.13 MB / 468 行）复制到快照目录
```

### 2.3 三级检查点（v1.3）

**核心问题**：单独文本备份不够，需要"计划→迭代→归档"完整闭环。

**解决方案**：三段式流程

```bash
# 阶段 1：开始任务 → 计划快照
node scripts/parallel/plan-snapshot.js "任务" "描述" "mermaid" "文件1,文件2"

# 阶段 2：worker 完成 → 迭代快照
node scripts/会话快照/save.js "标题" "标签" -m "下一步"

# 阶段 3：全部完成 → 全局归档
bash scripts/parallel/global-archive.sh "任务"
# 生成 archives/<任务>-<时间戳>/ 目录
# 含：snapshots-index.md + COMPLETION-REPORT.md + Git tag
```

**自动提示**：save.js 检测"完成/里程碑/交付"标签 → 启动段代码块自动加"💡 三级检查点提示"

### 2.4 Git Worktree 多工作区（P0）

**核心问题**：多 worker 并行时文件修改会冲突。

**解决方案**：每个 worker 独立 worktree 目录 + Git 分支

```bash
# 创建 2 个并行 worktree（在 .worktrees/ 下）
bash scripts/parallel/worktree-parallel.sh "任务名" 2

# 合并回主分支
bash scripts/parallel/worktree-merge.sh "任务名"
# 自动删除 worktree + 分支
# 创建 Git tag: archive-任务名-时间戳
```

**实测**：
- 串行：11 秒
- 并行：7 秒
- **提速 36%**

### 2.5 自我约束规范（v1.3.1）

**核心问题**：AI 改完代码后，你每次都得提醒"存快照"、"更新文档"。

**解决方案**：AI 完成后**自动**按规范执行收尾

**改动分级**：

| 级别 | 触发 | 自动动作 |
|:-----|:-----|:---------|
| 🟢 微小 | typo/注释 | 跳过 |
| 🟡 小 | bug fix/参数 | 测试 + 快照 + KB |
| 🔴 大 | 新功能/架构 | + 更新文档 |
| 🏁 里程碑 | v1.X 完成 | + 全局归档 |

**触发关键词**：
- 用户说"做完了/OK/完成/搞定" → AI 自动跑流程
- 用户没说但改了 ≥ 1 文件 → AI 主动跑（不询问）

**位置**：
- 完整规范：`scripts/orchestrator/自我约束规范.md`
- 自动加载：`.claude/rules/self-discipline.md`

### 2.6 上下文分片（v1.4）

**核心问题**：工作空间太大（2GB+），AI 每次都加载所有内容，浪费 token。

**解决方案**：`.claudeignore` 排除大目录

```
# 排除 12 个目录（2GB+）
AI-【0-4】/        ← 范式/参考/学习/项目
.Codex/             ← 其他 AI 工具
.ai-memory/          ← 跨 IDE 共享
.automation/         ← 自动化脚本
archives/            ← 归档
logs/                ← 日志
node_modules/        ← 依赖
.git/                ← Git 内部
.worktrees/          ← 临时 worktree
```

**效果**：AI 不主动读这些目录（除非你明确要求）→ 理论省 60% token

### 2.7 QA 子代理（v1.5）

**核心问题**：改完代码，谁来验证？

**解决方案**：只读 QA 子代理 + 斜杠命令

```bash
# 1. 派 QA 子代理
/qa  # 跑测试 + 写报告

# 2. 或手动
node scripts/orchestrator/test-dispatcher.js
node scripts/orchestrator/test-e2e.js
```

**子代理定义**：`.claude/agents/qa-reviewer.md`
- 模型：opus
- 只读不写
- 5 步流程：单元 + e2e + 钩子 + token + 集成
