## 第一部分：Claude Code 原生能力优化（6 个主流进阶优化）

### 1. 开启原生 fork 子进程

**真正的子代理，不是文字模拟，优先级最高**

Anthropic 内置了 `FORK_SUBAGENT` 分支能力，对应操作系统 fork 子进程，每个子代理拥有独立上下文，只把结果回传给主会话，不会污染主线对话。

| 项目         | 说明                                                         |
| :----------- | :----------------------------------------------------------- |
| **环境变量** | `CLAUDE_CODE_FORK_SUBAGENT=1`                                |
| **机制**     | 主代理 fork 出独立 worker，子进程只保留自己的任务快照，执行完毕只上交变更日志 + 代码，中间过程全部丢弃 |
| **最佳配比** | 同时并行 2-3 个 fork 子进程（和你当前规划完全匹配）；超过 4 个会争抢 token 预算，收益递减 |
| **配套**     | 给每个子进程分配独立 Git worktree 工作目录，文件修改互不冲突，天然支持多模块并行开发 |

> 💡 对比：你之前纯文本角色隔离，只是逻辑隔离；fork + worktree 是**运行时 + 文件双隔离**，稳定性提升一大截。


### 2. 上下文预算分片

**解决大项目越跑越慢，Token 砍掉 60%**

这是所有开源方案里必带的基础优化：

| 步骤 | 操作                                                         |
| :--- | :----------------------------------------------------------- |
| 1    | 在 `CLAUDE.md` 划定只读源码目录，强制排除 `node_modules`、构建产物、日志 |
| 2    | 划分静态缓存区（框架代码、公共配置，启用 Claude 提示词缓存，不用反复重传） |
| 3    | 动态任务区只留给当前子代理对应的模块代码                     |
| 4    | 自动压缩：上下文占用达到 50% 自动 compact，只保留计划 + 快照，清理中间闲聊 |

**行业标准写法**：静态内容 ≤ 8KB，留给代码生成的动态内容占 40% 额度。


### 3. 三级检查点快照

**把你现有的备份机制标准化**

现在社区已经把快照做成三段式闭环，替代单纯的文本备份：

| 级别         | 内容                                                         |
| :----------- | :----------------------------------------------------------- |
| **计划快照** | 任务拆分完成，保存方案 + Mermaid 原图                        |
| **迭代快照** | 每个子代理完成编码，输出 diff + 版本日志                     |
| **全局归档** | 所有子进程合并成果，自动把所有绘图源码写入历史目录，生成可回滚的 Git 提交点 |

配合会话断点保存，就算关掉窗口，下次可以从最近快照继续执行，不会重头再来。


### 4. 专职角色分工

**多代理专业化，而不是多个一模一样的 worker**

现代多代理不再是一堆相同编码进程，而是固定分工团队：

| 角色                        | 职责                           | 说明                         |
| :-------------------------- | :----------------------------- | :--------------------------- |
| **主调度器（Coordinator）** | 任务拆分、队列管理、结果合并   | 不写代码                     |
| **开发 Worker**             | 并行执行业务编码               | 2-3 个，带独立快照           |
| **QA 子代理**               | 代码自检 + 单元测试 + 漏洞扫描 | 独立并行执行，不占用编码线程 |

> 开源项目 `everything-claude-code` 里预制了 30 个专职子代理模板，开箱即用。


### 5. 任务依赖调度

**线程池 + 阻塞队列**

| 任务类型             | 调度策略                                   |
| :------------------- | :----------------------------------------- |
| **无依赖平级任务**   | 立刻分发到空闲子进程并行跑                 |
| **有前后依赖的任务** | 进入等待队列，形成简易线程池，避免无序执行 |

彻底杜绝串行排队，保证 CPU + 模型调用始终满载。


### 6. 后台异步执行

| 操作     | 效果                   |
| :------- | :--------------------- |
| `Ctrl+B` | 把子代理放到后台运行   |
| 主线程   | 同时安排下一批任务     |
| `/tasks` | 查看所有子进程运行状态 |

做到 **“一边编码、一边测试、一边写文档”三线推进**。

## 第四部分：关键避坑（社区踩过最多的雷）

| 序号 | 避坑点                        | 说明                                                         |
| :--: | :---------------------------- | :----------------------------------------------------------- |
|  1   | **不要开超过 3 个并行子代理** | 单会话上下文上限摆在这，4 个及以上 worker 会互相挤占指令空间，备份、快照规则容易被遗忘，提速不升反降；2-3 个是性价比黄金区间 |
|  2   | **文本模拟 ≠ fork 子进程**    | 纯靠角色标签隔离，上下文依然互相交织，轮次一多就会串任务；一定要开启原生 fork，子进程中间日志自动丢弃，只留存快照成果 |
|  3   | **快照不要写进主上下文**      | 子代理只把版本变更 + 图纸代码回传主线，完整日志保存在本地文本，避免占用会话 token |
